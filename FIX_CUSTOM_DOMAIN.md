# Исправление ошибки кастомного домена

## Ошибка
```
The custom domain `artsercheklist` is not properly formatted.
```

## Причина
В настройках GitHub Pages указан неправильный формат кастомного домена. Домен должен быть в формате полного доменного имени (например, `example.com`), а не просто строкой.

## Решение

### Вариант 1: Удалить кастомный домен (рекомендуется)

Если вам не нужен кастомный домен, просто удалите его:

1. Перейдите в **Settings** → **Pages**
2. Найдите раздел **"Custom domain"**
3. Нажмите кнопку **"Remove"** (красная кнопка справа от поля ввода)
4. Подтвердите удаление

После этого GitHub Pages будет работать на стандартном адресе:
```
https://Artser.github.io/ChatList/
```

### Вариант 2: Исправить формат домена

Если вам нужен кастомный домен, используйте правильный формат:

1. Перейдите в **Settings** → **Pages**
2. В разделе **"Custom domain"** введите правильный домен:
   - ✅ Правильно: `artsercheklist.com` или `www.artsercheklist.com`
   - ❌ Неправильно: `artsercheklist` (без точки и расширения)
3. Нажмите **Save**

**Важно:** Для использования кастомного домена нужно:
- Иметь зарегистрированный домен
- Настроить DNS записи (A, CNAME)
- Подождать распространение DNS (может занять до 24 часов)

## Рекомендация

Для начала работы с GitHub Pages **рекомендуется удалить кастомный домен** (Вариант 1), чтобы использовать стандартный адрес `Artser.github.io/ChatList/`.

Кастомный домен можно настроить позже, когда будет готов реальный домен.

## После исправления

1. Удалите или исправьте кастомный домен
2. Запустите workflow заново:
   - **Actions** → **Deploy GitHub Pages** → **Run workflow**
3. Подождите 1-2 минуты
4. Проверьте результат в **Actions**
5. Кнопка **"Visit Site"** появится в **Settings** → **Pages**
