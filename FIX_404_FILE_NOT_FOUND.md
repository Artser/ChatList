# Исправление ошибки 404 "File not found" на GitHub Pages

## Проблема
GitHub Pages работает, но показывает ошибку 404 "File not found". Это означает, что файл `index.html` не найден на сайте.

## Возможные причины

1. **Workflow не развернул файлы** - нужно перезапустить
2. **Файлы развернуты в неправильную папку** - нужно проверить путь
3. **Нужно подождать распространения** - GitHub Pages обновляется не мгновенно

## Решение

### Шаг 1: Проверьте последний workflow run

1. Перейдите в **Actions**
2. Найдите последний run для "Deploy GitHub Pages"
3. Убедитесь, что он завершился успешно (зеленый ✅)
4. Если failed (красный ❌) - исправьте ошибки

### Шаг 2: Перезапустите workflow

1. Перейдите в **Actions** → **Deploy GitHub Pages**
2. Нажмите **"Run workflow"** → **"Run workflow"**
3. Подождите 2-3 минуты

### Шаг 3: Проверьте развертывание

1. После успешного завершения workflow
2. Подождите еще 1-2 минуты (GitHub Pages обновляется не мгновенно)
3. Обновите страницу сайта (Ctrl+F5)
4. Проверьте, что файл доступен

### Шаг 4: Проверьте путь к файлу

Убедитесь, что workflow правильно загружает файлы:

1. Откройте последний успешный run
2. Найдите шаг **"Upload artifact"**
3. Проверьте, что путь указан как `./docs`
4. Убедитесь, что файл `index.html` находится в папке `docs/`

## Альтернативное решение: Проверьте структуру файлов

Убедитесь, что структура правильная:

```
ChatList/
└── docs/
    └── index.html  ← должен быть здесь
```

## Если проблема не решается

1. **Проверьте логи workflow:**
   - Actions → последний run → шаг "Upload artifact"
   - Убедитесь, что файлы загружены

2. **Проверьте настройки Pages:**
   - Settings → Pages
   - Source должен быть "GitHub Actions"
   - Должна быть информация о последнем развертывании

3. **Попробуйте очистить кэш:**
   - Обновите страницу с Ctrl+F5 (жесткая перезагрузка)
   - Или откройте в режиме инкогнито

4. **Проверьте URL:**
   - Правильный URL: `https://Artser.github.io/ChatList/`
   - Неправильный: `https://Artser.github.io/ChatList/docs/` или другие варианты

## Быстрая проверка

После перезапуска workflow:
1. Подождите 2-3 минуты
2. Откройте: `https://Artser.github.io/ChatList/`
3. Должна открыться страница с лендингом

Если все еще 404:
- Проверьте логи workflow
- Убедитесь, что файл `docs/index.html` закоммичен
- Попробуйте перезапустить workflow еще раз
