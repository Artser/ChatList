# Удаление кастомного домена для работы GitHub Pages

## Проблема
Кастомный домен `artsercheklist.com` указан, но DNS записи не настроены, что блокирует развертывание GitHub Pages.

## Решение: Удалить кастомный домен

### Пошаговая инструкция:

1. **Откройте настройки Pages**
   - Перейдите в **Settings** → **Pages**
   - Или по прямой ссылке: https://github.com/Artser/ChatList/settings/pages

2. **Найдите раздел "Custom domain"**
   - Прокрутите вниз до раздела "Custom domain"
   - Вы увидите поле с текстом `artsercheklist.com`

3. **Удалите кастомный домен**
   - Нажмите красную кнопку **"Remove"** справа от поля ввода
   - Подтвердите удаление (если потребуется)

4. **Сохраните изменения**
   - После удаления настройки сохранятся автоматически

5. **Проверьте результат**
   - Раздел "Custom domain" должен стать пустым
   - Ошибки DNS должны исчезнуть

## После удаления кастомного домена

1. **GitHub Pages будет работать на стандартном адресе:**
   ```
   https://Artser.github.io/ChatList/
   ```

2. **Запустите workflow заново:**
   - Перейдите в **Actions**
   - Выберите "Deploy GitHub Pages"
   - Нажмите **"Run workflow"** → **"Run workflow"**

3. **Подождите 1-2 минуты** и проверьте результат

4. **Кнопка "Visit Site" появится** в Settings → Pages после успешного развертывания

## Если позже захотите настроить кастомный домен

1. Зарегистрируйте домен у регистратора
2. Настройте DNS записи:
   - A запись: `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - Или CNAME запись: `Artser.github.io`
3. Подождите распространения DNS (до 24 часов)
4. Добавьте домен в Settings → Pages → Custom domain

## Важно!

**Пока кастомный домен не настроен правильно, он блокирует работу GitHub Pages.**

Удалите его сейчас, чтобы GitHub Pages заработал на стандартном адресе.
